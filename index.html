자동화의 3원칙

자동화는 복잡한 코딩이 아닙니다. 아래 3가지만 기억하세요.

1. Trigger (방아쇠)

"언제 시작해?" (예: Tally 폼이 제출되면)

2. Action (행동)

"무슨 일을 해?" (예: 메일을 보내)

3. Connection (연결)

"어떤 도구로?" (예: 지메일, Airtable)

오늘의 목표 시나리오

📝 Tally 폼 작성 (Input)
🗂️ Airtable 저장 (DB)
🪄 수료증 자동 생성 & 발송

AI와 함께하는 자동화 기획

무작정 만들기 전에 "무엇을, 어떻게" 자동화할지 AI와 상의하는 단계입니다.

1

자동화할 업무 찾기

모든 업무를 자동화할 수는 없습니다. 아래 기준에 맞는 업무를 찾아보세요.

✅ 자동화 가능 (규칙이 있음) "신청서가 들어오면 → Airtable에 기록하고 → 메일을 보낸다."
❌ 자동화 불가능 (판단이 필요함) "신청 사연을 읽어보고 → 감동적이면 합격시킨다."
2

AI에게 기획 맡기기

초보자 맞춤형 프롬프트입니다. AI가 여러분의 상황을 이해하고 가장 쉬운 방법을 제안하도록 설계되었습니다.

"나는 [수강신청 -> 관리 -> 수강증 발급] 업무를 자동화하고 싶어. 이것을 Make.com에서 구현하려고 해."

# 요청사항

  • Make.com 초보자니까 최소 기능(MVP)으로 설계해줘.
  • 비개발자도 이해하기 쉬운 용어로 설명해줘.
  • 바로 답을 주지 말고, 구현에 필요한 정보가 부족하면 나에게 먼저 질문해줘. 내 답변을 듣고 나서 전체 시나리오를 짜줘.

Tip: 대괄호 [ ] 안의 내용만 내 업무로 바꾸세요!

설계: 종이에 그려보기

바로 컴퓨터를 켜지 마세요. 손으로 그릴 수 있어야 만들 수 있습니다.

설계의 3단계

1

입력(Input) 정의

어떤 데이터가 들어오는가? (예: 이름, 이메일)

2

처리(Process) 순서

데이터가 어디를 거쳐가는가? (폼 → Airtable → 메일)

3

출력(Output) 확인

최종 결과물은 무엇인가? (PDF 수료증)

실습 체크포인트

마법의 순간: 수료증 자동 발급

데이터를 가져와서 디자인 템플릿에 넣고, PDF로 만들어 이메일까지 보냅니다.

Step 5-1. 구글 닥스 템플릿 준비

구글 닥스 문서 중앙에 아래 태그를 정확히 입력하세요. 이 부분이 이름으로 바뀝니다.

{{NAME}}

Step 5-2. 시나리오 확장하기

1

Google Docs Create a Document from a Template

- Template ID: 방금 만든 문서 선택
- {{NAME}}: Airtable의 'Name' 필드 매핑

2

Google Drive Download a File

- File ID: 위에서 생성된 문서 ID 매핑
- Type: PDF Document (.pdf) 선택

3

Gmail Send an Email

- To: 신청자 이메일 주소 (Airtable)
- Attachments: 방금 다운로드한 PDF 데이터 선택
